# Contributing

Ion aims to stay minimal. Contributions that simplify, fix bugs, improve docs, or add examples are welcome.

## Workflow

1. Open an issue describing the change you'd like to make
2. Once discussed, fork the repo and make your changes
3. Ensure CI passes: `uv run ruff check`, `uv run pyright`, `uv run pytest`
4. Open a PR against `main`

For trivial fixes (typos, small doc corrections), a PR without an issue is fine.
